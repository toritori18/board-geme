# Git運用ルール

## 基本方針

- `main` への直接コミット・プッシュは禁止
- **機能追加** は `feature/<機能名>` ブランチで作業し、PR 経由で main へマージする
- **バグ修正** は `fix/<バグ名>` ブランチで作業し、完了後に main へ直接プッシュする（PR 不要）

## コミット & プッシュの手順

### 機能追加

1. feature ブランチを作成して作業する
   ```
   git checkout -b feature/<機能名>
   ```

2. 変更をステージング・コミットする
   ```
   git add <変更したファイル>
   git commit -m "feat: 変更内容の説明"
   ```

3. ブランチをプッシュして PR を作成する
   ```
   git push origin feature/<機能名>
   ```

### バグ修正

1. fix ブランチを作成して作業する
   ```
   git checkout -b fix/<バグ名>
   ```

2. 変更をステージング・コミットする
   ```
   git add <変更したファイル>
   git commit -m "fix: 変更内容の説明"
   ```

3. 修正完了後、main へマージしてプッシュする
   ```
   git checkout main
   git merge fix/<バグ名>
   git push origin main
   ```

## ブランチ運用

- `main` ブランチ: 本番相当の安定コード
- `feature/<機能名>`: 機能追加用（PR 経由でマージ）
- `fix/<バグ名>`: バグ修正用（完了後に main へ直接プッシュ）

## コミットメッセージ規約

| プレフィックス | 用途 |
|---|---|
| `feat:` | 新機能の追加 |
| `fix:` | バグ修正 |
| `refactor:` | リファクタリング |
| `docs:` | ドキュメント変更 |
| `chore:` | ビルド・設定変更など |

例: `feat: ゲームボードの初期表示を追加`

## 注意事項

- `.env` やシークレット情報を含むファイルは絶対にコミットしない
- `git push --force` は原則禁止（main ブランチへは特に厳禁）
- プッシュ前に `git diff` や `git status` で変更内容を確認する
